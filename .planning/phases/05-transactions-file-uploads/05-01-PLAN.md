Create `backend/src/routes/transactions.ts` exporting `transactionsRouter`:

**Zod schemas:**
- `CreateTransactionSchema`: clientId (uuid string), description (optional string), deliveredAt (optional ISO string), initialPayment (string regex /^\d+(\.\d{1,2})?$/, default "0"), clientNotes (optional string), internalNotes (optional string), items array of { productId (optional uuid), description (string min 1 max 255), quantity (string regex /^\d+(\.\d{1,3})?$/ min "0.001"), unitPrice (string regex /^\d+(\.\d{1,2})?$/) } with min 1 item.
- `ApproveSchema`: (no body needed)
- `RejectSchema`: { reason: string min 1 max 500 }

**Helper functions (in same file):**
- `toCents(str: string): number` — `Math.round(parseFloat(str) * 100)`
- `fromCents(cents: number): string` — `(cents / 100).toFixed(2)`
- `async nextReferenceNumber(tx: typeof db, companyId: string): Promise<string>` — upsert companyCounters with `onConflictDoUpdate` setting `lastSequence = lastSequence + 1`, then select to read new value. Format: `TXN-${year}-${seq.padStart(4, '0')}`.

**Endpoints:**

1. `POST /` — Create transaction. Parse `req.body.data` (JSON string from multipart — parsed by multer middleware, but if no files, parse `req.body` directly). Validate with CreateTransactionSchema.
   - Wrap everything in `db.transaction(async (tx) => { ... })`.
   - Generate referenceNumber via nextReferenceNumber(tx, companyId).
   - Compute totalAmount server-side: sum line items using integer cents arithmetic: `items.reduce((acc, item) => acc + Math.round((Math.round(parseFloat(item.quantity) * 1000) * toCents(item.unitPrice)) / 1000), 0)`.
   - Set status: `req.user!.role === 'owner' ? 'active' : 'pending_approval'` (FR-05.6, FR-05.7).
   - Insert transaction row, then insert all transactionItems (with productId if provided, else null).
   - If status === 'active' and totalCents > initialPaymentCents: insert debt with totalAmount = fromCents(totalCents - initialPaymentCents), status 'open' (FR-05.5).
   - If status === 'pending_approval': find all users with role='owner' in same company, insert one notification per owner with entityType='transaction', action='submitted_for_approval'.
   - Insert audit_log entry with action 'transaction.created'.
   - Return 201 with created transaction + items.

2. `GET /` — List transactions for company. Support query params: search (ilike on referenceNumber or description), clientId (uuid filter), status (exact match on txStatusEnum), dateFrom/dateTo (filter on deliveredAt). Always scope by `eq(transactions.companyId, req.companyId!)`. Join with clients for client name, join with users (createdBy) for submitter name. Order by createdAt DESC.

3. `GET /:id` — Get single transaction by ID scoped to company. Include transactionItems (join or subquery). Include documents associated with entityType='transaction' and entityId=id. Include client name and submitter name via joins. If `req.user!.role === 'client'`, exclude internalNotes from response (set to undefined/null) per FR-05.12.

4. `POST /:id/approve` — Owner only. Wrap in `db.transaction(async (tx) => { ... })`:
   - SELECT transaction with `.for('update')` where id=id AND companyId=companyId (FR-05.7).
   - If not found or status !== 'pending_approval', return 400 `{ error: 'Transaction is not pending approval' }`.
   - Update status to 'active', set approvedBy=req.user!.sub, approvedAt=new Date().
   - If totalAmount > initialPayment (cents comparison): insert debt.
   - Insert notification for createdBy user with action='approved'.
   - Insert audit_log with action 'transaction.approved'.
   - Return 200 with updated transaction.

5. `POST /:id/reject` — Owner only. Validate body with RejectSchema.
   - SELECT transaction with `.for('update')`.
   - If status !== 'pending_approval', return 400.
   - Update status to 'draft' (FR-05.9).
   - Insert notification for createdBy user with action='rejected', store reason in notification or in a separate field.
   - Insert audit_log with action 'transaction.rejected'.
   - Return 200.

6. `POST /:id/void` — Owner only. Update status to 'voided', set voidedBy, voidedAt, voidReason from body. Return 200.

Create `backend/src/routes/notifications.ts` exporting `notificationsRouter`:

1. `GET /unread-count` — `SELECT COUNT(*) FROM notifications WHERE recipientId = req.user!.sub AND isRead = false AND companyId = req.companyId!`. Return `{ count: number }`.

2. `GET /` — List notifications for current user, ordered by createdAt DESC, limit 50. Join with transactions (via entityId) for referenceNumber, totalAmount. Join with clients for client name. Join with users (createdBy on the transaction) for submitter name. Return array with: id, entityType, entityId, action, isRead, createdAt, transaction.referenceNumber, transaction.totalAmount, client.fullName, submitter.fullName.

3. `PATCH /:id/read` — Update notification isRead=true where id=id AND recipientId=req.user!.sub. Return 204.

4. `POST /mark-all-read` — Update all unread notifications for current user to isRead=true. Return 204.

**Mount in app.ts:**
- `app.use('/api/v1/transactions', authenticate, requireTenant, requireRole('owner', 'collaborator', 'viewer'), transactionsRouter);` — all tenant roles can view; POST handler internally checks role for status assignment.
- `app.use('/api/v1/notifications', authenticate, requireTenant, notificationsRouter);` — all roles (owner/collaborator need notifications; viewer is harmless).

Place mount lines BEFORE the 404 handler, after the existing `/api/v1/products` mount.
