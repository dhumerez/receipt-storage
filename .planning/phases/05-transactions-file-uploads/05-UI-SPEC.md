## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge inline padding, remove [×] button padding |
| sm | 8px | Input inner padding, row gap between form fields (`gap-2`) |
| md | 16px | Default element spacing, form field padding (`px-4 py-2`), section content padding |
| lg | 24px | Section margin between form groups (`mb-6`) |
| xl | 32px | Page padding (`p-8` = 32px on all sides — established pattern) |
| 2xl | 48px | Slide-out panel section separator, empty state vertical padding (`py-16` = 64px, acceptable exception) |
| 3xl | 64px | Not directly used; reserved for page-level breathing room |

Exceptions:
- Touch targets: minimum 44px height on all interactive rows and nav items — established via `min-h-[44px]` in Sidebar.tsx and ProductTableRow.tsx. Apply to: bell icon button, notification panel items, file attachment buttons ("Take Photo", "Choose from Gallery"), line item rows, transaction table rows.
- Slide-out notification panel: fixed width 384px (24rem) — not a spacing token, but a layout constraint.

## Color

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `gray-50` (#F9FAFB) | Page background (`bg-gray-50`) | AppLayout.tsx — `className="min-h-screen bg-gray-50 flex"` |
| Secondary (30%) | `white` (#FFFFFF) | Sidebar, cards, modal surfaces, table container, slide-out panel | Sidebar: `bg-white`; ProductTable: `border border-gray-200 rounded-lg`; ProductModal: `bg-white rounded-lg shadow-xl` |
| Accent (10%) | `blue-600` (#2563EB) | See reserved list below | Established in Phases 3–4 |
| Destructive | `red-600` / `red-700` | Reject action only — inline reject confirm button in notification panel | Pattern: `bg-red-50 border border-red-200 text-red-700` for error banners; `text-red-700` for destructive text |

Accent (`blue-600`) reserved for:
1. Primary CTA buttons — "Save Transaction", "Add to Line Items" (catalog picker confirm), "Approve" (notification panel)
2. Active navigation state — sidebar `border-l-4 border-blue-600 bg-blue-50 text-blue-600` (existing pattern, do not change)
3. Focus rings — `focus:ring-2 focus:ring-blue-500` on all inputs (existing pattern)
4. Link text — "← Clients" back-link pattern; client name links on transaction detail
5. Unread notification badge background — `bg-blue-600 text-white` dot/count on bell icon
6. Inline hover state on clickable price cells — `hover:bg-blue-50` (existing ProductTableRow pattern)

NOT accent: table row hover (use `hover:bg-gray-50`), filter toggles (use `gray-100`/`gray-200`), status badges for pending/active/voided.

| `active` | `green-100` | `green-800` |

## Interaction Contracts

### Bell Icon + Notification Panel

- Bell icon position: top-right of AppLayout header, to the left of a user menu placeholder or as the rightmost header element.
- Bell renders for roles: `owner` and `collaborator` (hidden for `viewer` and `client`).
- Unread badge: circular `bg-blue-600 text-white` dot, 18px × 18px, positioned `absolute -top-1 -right-1` relative to bell wrapper. Shows count if ≤ 9; shows "9+" if > 9. Hidden when count is 0.
- Panel open trigger: click bell icon. Panel slides in from the right. Overlay: `fixed inset-0 bg-transparent` to capture outside clicks and close.
- Panel close: click outside panel area, or click bell again, or click an explicit "×" close button in panel header.
- Panel animation: `translate-x-0` → `translate-x-full` using `transition-transform duration-200 ease-in-out` (CSS only, no animation library).
- Panel width: `w-96` (384px) on desktop; full-width `w-full` on mobile (< 640px breakpoint).
- Panel z-index: `z-50` (same as existing modals — panel and modal should not coexist).

### Reject Inline Expansion

- Click "Reject": the button transforms — row expands to show a `<textarea rows={2}>` labeled "Reason for rejection" and a "Confirm Reject" button below it.
- "Reject" button hidden during expansion (replaced by "Confirm Reject" and a "Cancel" link to collapse).
- Textarea: `placeholder="Describe why this transaction is being rejected"`, `maxLength={500}`, `required`.
- "Confirm Reject" button: `bg-red-600 text-white hover:bg-red-700` — this is the only use of red as a button background in Phase 5. All other destructive actions use text links or are non-destructive by design.
- If reject reason is empty: show inline validation text "A reason is required to reject a transaction." below the textarea; do not submit.

### Line Item Builder

- Each row: `grid grid-cols-[1fr_80px_100px_100px_32px]` on desktop; collapses to stacked layout on mobile (< 640px).
- Columns: Name | Qty | Unit Price | Line Total | [×]
- Catalog item Name field: `bg-gray-50` background, `cursor-default` — visually read-only but text is selectable.
- Free-form Name field: standard white background, editable.
- Line Total: right-aligned, formatted as currency string (`$X.XX`), computed on every qty/unit-price change.
- Running total below all rows: "Total: $X.XX" in `text-sm font-semibold text-gray-900`, right-aligned.
- Add buttons container: `flex gap-2 mt-3`; both buttons always visible.
- `[+ Catalog]` button: `border border-gray-300 text-gray-700 hover:bg-gray-50` (secondary style — not accent).
- `[+ Free-form]` button: `border border-gray-300 text-gray-700 hover:bg-gray-50` (same secondary style).

### File Attachment Section

- Drop zone: dashed border `border-2 border-dashed border-gray-300 rounded-lg`, `bg-gray-50`, min-height 80px, drag-active state switches to `border-blue-500 bg-blue-50`.
- "Take Photo" button: `border border-gray-300 rounded-md px-4 py-2 text-sm text-gray-700 hover:bg-gray-50`. Hidden `<input type="file" accept="image/*" capture="environment">`.
- "Choose from Gallery" button: same visual style. Hidden `<input type="file" accept=".jpg,.jpeg,.png,.webp,.pdf,.heic" multiple>`. No `capture` attribute.
- Both buttons always visible; never hidden or disabled based on device.
- File thumbnails: `w-20 h-20 rounded-md object-cover` for images. PDF files: gray `w-20 h-20 rounded-md bg-gray-100 flex items-center justify-center` with a document SVG icon and truncated filename below.
- Remove [×]: `absolute -top-1 -right-1 w-5 h-5 rounded-full bg-gray-700 text-white text-xs flex items-center justify-center hover:bg-gray-900` — positioned relative to thumbnail wrapper.
- Thumbnail grid: `flex flex-wrap gap-3 mt-3`.
- File count indicator: "3 / 5 files" in `text-xs text-gray-500` below the grid when files are staged.

### New Transaction Form Layout

- Full page at `/transactions/new`; page padding `p-8` (32px) matching established pattern.
- Page heading: "New Transaction" at `text-2xl font-semibold text-gray-900`.
- Back / cancel: "Discard Changes" as a text button `text-sm text-gray-600 hover:text-gray-900` to the left of the "Save Transaction" primary button. Both in a footer bar `sticky bottom-0 bg-white border-t border-gray-200 px-8 py-4 flex justify-end gap-3` — keeps form actions visible while scrolling.
- Form sections stacked vertically with `space-y-8` between them.
- Each section: optional `<h2 className="text-sm font-semibold text-gray-700 mb-3">` label above the fields.
- Client selector: standard `<select>` or a searchable combobox with `border border-gray-300 rounded-md` styling — matches input pattern.
- All inputs: `w-full px-3 py-2 border border-gray-300 rounded-md focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-blue-500 text-sm` — established pattern from ProductModal.tsx.
- Textareas: `resize-none` — established pattern.
- Delivery date: `<input type="date">` with same input class.
- Initial payment: number input with `min="0" step="0.01" placeholder="0.00"`.
- Internal notes section: only visible to owner and collaborator roles. Render conditionally based on auth role.

### Transaction Detail Page Layout

- Same `p-8` page padding.
- Back link: "← Transactions" same pattern as "← Clients" in ClientDetailPage.
- Section separator: `border-t border-gray-200 mt-8 pt-8` between sections.
- Attachment thumbnails: same thumbnail grid as FileAttachmentSection, but thumbnails are links (`<a href>` that triggers authenticated download) — no [×] remove on detail view.
- Status badge: `TransactionStatusBadge` inline in header, after reference number.
- Internal notes section: only rendered if `req.user.role` is `owner` or `collaborator` (not for viewer; portal has its own endpoint that structurally excludes internalNotes).

### Transaction List Page

- Same page layout as ProductsPage: `p-8`, heading + button row, filter row, table or empty state.
- Status filter: 4 options — "All", "Pending", "Active", "Voided". Render as a segmented button group (same visual as StatusFilterToggle but with 4 segments instead of 3).
- Date range filter: two `<input type="date">` fields labeled "From" and "To", side by side.
- Client filter: `<select>` dropdown loaded from the clients API.
- Filter row: `flex flex-wrap gap-2 mb-4` — responsive, wraps on mobile.
