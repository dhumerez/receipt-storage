## Interaction Contracts

### Inline Price Edit Cell (D-03)

Display mode:
- Cell renders: `$` + `parseFloat(unitPrice).toFixed(2)` (e.g., `$12.50`)
- Visual cue: `cursor-pointer` on the cell; no explicit edit icon — clicking the value itself activates edit mode
- Cell padding matches other cells: `px-4 py-3`

Edit mode (activated by click):
- Cell becomes a controlled `<input type="number" min="0" step="0.01">` with `autoFocus`
- Input width: `w-24` (fits up to 6-digit prices); no border radius change; `border border-gray-300 rounded px-2 py-1`
- Save triggers on: Enter key OR blur
- Cancel (discard): Escape key — reverts to original value, exits edit mode
- Double-save guard: `savedRef = useRef(false)` — set before mutation.mutate(); reset in onSettled
- Invalid input (NaN, negative): revert to original value silently on blur/Enter; no toast
- Network error on save: revert to original value; show inline row-level error text in red-50 banner (same pattern as ClientModal error div)
- Pending state: `disabled` on input + `opacity-50`; no spinner needed for quick network ops

### Edit Icon (D-02)

- Pencil icon button at end of each row (rightmost column, "Actions")
- Icon: inline SVG pencil, `w-4 h-4 text-gray-400 hover:text-gray-600`
- Button: `p-1 rounded hover:bg-gray-100`; accessible `aria-label="Edit {product name}"`
- Click opens ProductModal in edit mode, pre-filled with all product fields

### Deactivate / Reactivate (D-04)

- Active product row: Deactivate button/link — opens DeactivateProductModal (follows DeactivateConfirmModal pattern)
- Inactive product row: "Reactivate" button — calls `PATCH /api/v1/products/:id/reactivate` directly (no confirmation modal needed; reversible action)
- Reactivate: `text-sm text-blue-600 hover:underline` inline button; no modal
- Both actions follow the same row-level pattern; no page navigation after either action

### Search Debounce

300ms debounce on search input — copy the `useRef<ReturnType<typeof setTimeout>>` pattern from ClientsPage.tsx verbatim.

### Modal Behavior

All modal behavior matches ClientModal.tsx exactly:
- Backdrop click closes modal: `if (e.target === e.currentTarget) onClose()`
- Escape key closes modal: `addEventListener('keydown', handler)` with cleanup
- First input (`name`) receives focus on open via `useRef` + `setTimeout(..., 50)`
- Submit button disabled while `mutation.isPending`

## Form Field Contracts

### ProductModal fields

| Field | Type | Required | Validation | Placeholder |
|-------|------|----------|-----------|-------------|
| Name | `<input type="text">` | Yes | min 1 char, max 255 | — |
| Unit Price | `<input type="number" min="0" step="0.01">` | Yes | non-negative number, ≤ 2 decimal places | "0.00" |
| Unit of Measure | `<input type="text">` | No | max 50 chars | "e.g., box, unit, kg" |
| Description | `<textarea rows={3}>` | No | no max enforced in UI | — |

Unit Price note: frontend sends as `parseFloat(value).toFixed(2)` string to match backend Zod `z.string().regex(/^\d+(\.\d{1,2})?$/)`. Never use `type="number"` to submit directly — always `.toFixed(2)` before mutation.

## Accessibility Contracts

- All interactive table rows: `role="button" tabIndex={0}` with `onKeyDown` Enter/Space handler — mirrors ClientTableRow.tsx
- Edit icon buttons: `aria-label="Edit {product.name}"` (product name interpolated)
- Status badge: `aria-label="Status: Active"` or `aria-label="Status: Inactive"` — mirrors ClientStatusBadge.tsx
- Table headers: `<th scope="col">` implicit in existing pattern; maintain
- Modal: backdrop div is a click target only; modal content div gets focus management via `firstInputRef`
- Inline price edit input: `aria-label="Unit price for {product.name}"` when in edit mode
