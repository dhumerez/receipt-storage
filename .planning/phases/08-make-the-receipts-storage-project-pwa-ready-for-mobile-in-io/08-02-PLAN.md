<objective>
Harden the iOS PWA surface: add iOS-specific meta tags to index.html, update viewport to include viewport-fit=cover for notch safety, remove the manual SW registration script (plan 01's injectRegister:'auto' replaces it), fix manifest.webmanifest start_url/scope to absolute paths, and add an automated DOM test verifying all iOS meta tags.

Purpose: iOS Safari ignores most of the Web App Manifest. Without apple-mobile-web-app-capable, iPhones do not enter standalone mode. Without viewport-fit=cover, content is clipped by the notch on iPhone 12+. Without the correct start_url and scope in the manifest, Lighthouse (and Chrome DevTools) report "SW does not control start_url" blocking installability on Android.

Output: index.html ready for iOS home screen installation; manifest.webmanifest fully correct for both iOS and Android; passing unit test confirming meta tags.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iOS Safari → standalone PWA | iOS creates separate storage partition; cookie isolation is a platform limitation |
| manifest.webmanifest → browser install | Manifest content controls what the OS displays for the installed app |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-05 | Spoofing | manifest.webmanifest start_url "." vs "/" | mitigate | Fixed to `start_url: "/"` and `scope: "/"` — SW at /sw.js controls scope "/" which matches; prevents Chrome DevTools "SW does not control start_url" error that blocks install on Android |
| T-8-06 | Auth Bypass (unintentional) | iOS cookie isolation after home screen install | accept | Platform limitation — cannot be mitigated in code. iOS performs one-time cookie copy at install time (iOS 17.2+). Document that re-authentication is expected on first standalone launch. The existing apiClient 401 interceptor + refresh flow handles this correctly. |
| T-8-07 | Tampering | Stale manual SW registration overriding vite-plugin-pwa generated SW | mitigate | Inline `navigator.serviceWorker.register()` script removed from index.html — eliminates race condition between stub and generated SW (RESEARCH.md Pitfall 1) |
</threat_model>
