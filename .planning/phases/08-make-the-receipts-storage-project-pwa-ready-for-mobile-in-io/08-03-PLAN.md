<objective>
Add two exact-match location blocks to nginx/receipts.conf that serve sw.js and manifest.webmanifest with Cache-Control: no-store headers. These blocks must appear before the existing catch-all location / block.

Purpose: The browser uses a byte-diff check on sw.js on every page load to detect service worker updates. If Nginx caches sw.js (even for 60 seconds), a deployed update will not reach users until the cache expires — breaking the entire SW update flow. manifest.webmanifest with a stale cache means the browser may display the old app name, scope, or icon set. no-store is the strictest directive: it prevents both browser and proxy caching entirely.

Output: Updated nginx/receipts.conf ready to be deployed to the Hetzner host. The file serves as the deployment source of truth — it must be copied to /etc/nginx/conf.d/receipts.conf on the server.
</objective>

```nginx
# /etc/nginx/conf.d/receipts.conf
# DEPLOYMENT: copy this file to /etc/nginx/conf.d/receipts.conf on the Hetzner host.
# Replace receipts.yourdomain.com with the actual subdomain BEFORE deploying.
# Run: sudo nginx -t && sudo nginx -s reload
# Do NOT modify restaurant.conf or any other existing config file.

# ── Step 1 (initial deploy): HTTP-only block ──────────────────────────────────
# Deploy this block first, reload Nginx, run certbot, then Step 2 takes over.
server {
    listen 80;
    server_name receipts.yourdomain.com;
    return 301 https://$host$request_uri;
}

# ── Step 2 (after certbot): HTTPS block ──────────────────────────────────────
server {
    listen 443 ssl;
    server_name receipts.yourdomain.com;

    # Let's Encrypt certificates (certbot --nginx populates these)
    ssl_certificate     /etc/letsencrypt/live/receipts.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/receipts.yourdomain.com/privkey.pem;
    include             /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam         /etc/letsencrypt/ssl-dhparams.pem;

    # File upload limit — multer limit is 10 MB; 12 m gives multipart overhead room
    client_max_body_size 12m;

    # API routes — proxy to Express on port 4000
    location /api/ {
        proxy_pass         http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }

    # ── PWA: Service worker — NEVER cache ────────────────────────────────────
    # The browser fetches sw.js on every page load for byte-diff update detection.
    # If cached (even briefly), SW updates never reach users after deployment.
    # "always" flag applies the header even on error responses (e.g., 502, 304).
    location = /sw.js {
        proxy_pass         http://127.0.0.1:4001;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Forwarded-Proto $scheme;
        add_header         Cache-Control "no-store, no-cache, must-revalidate, proxy-revalidate, max-age=0" always;
        expires            off;
    }

    # ── PWA: Web App Manifest — no-store for fresh manifest on every page load ─
    # Stale manifest = old app name, icons, or scope reaching the browser.
    # Scope mismatch between manifest and SW causes installability failures.
    location = /manifest.webmanifest {
        proxy_pass         http://127.0.0.1:4001;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Forwarded-Proto $scheme;
        add_header         Cache-Control "no-store, no-cache, must-revalidate, max-age=0" always;
        expires            off;
    }

    # React SPA — proxy to Vite/Node server on port 4001
    # All non-API routes handled here; SPA handles its own client-side routing
    location / {
        proxy_pass         http://127.0.0.1:4001;
        proxy_http_version 1.1;
        proxy_set_header   Host $host;
        proxy_set_header   X-Forwarded-Proto $scheme;
    }
}
```

| Threat ID | Category | Component | Disposition | Mitigation Plan |
