## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 12px | 400 (regular) | 1.5 | `text-xs uppercase tracking-wider` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Page Title | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |

Only 2 font weights used: 400 (regular) and 600 (semibold). Label differentiates from Body via its smaller 12px size and uppercase tracking. Page Title differentiates from Heading via its larger 24px size.

Table headers use: `text-xs uppercase tracking-wider text-gray-500` (established in ClientTable, ProductTable, TransactionTable). Weight is 400 (regular) — the uppercase tracking provides sufficient visual distinction.

Source: Codebase scan. EmptyState uses `text-xl font-semibold` for headings and `text-sm` for body. Sidebar uses `text-base font-semibold` for brand name, `text-sm` for nav items.

## Layout Contract

### Reports Page (`/reports`)

**Focal point:** The report table is the primary visual anchor. "Export PDF" is the primary action focal point, positioned at the bottom-right of the content area to conclude the data-reading flow.

```
+----------------------------------------------+
| [Company Report]  [Client Report]   (tabs)   |
+----------------------------------------------+
| Filter Bar:                                   |
| [Date From] [Date To] [Status v] [x Settled] |
+----------------------------------------------+
| Report Table                    (focal point) |
| Client Name | Total Debts | Paid | Balance   |
| (sortable)  | (sortable)  | ...  | (sorted)  |
|                                               |
| ...rows...                                    |
+----------------------------------------------+
| [Export PDF]             (action focal point)  |
+----------------------------------------------+
```

- Tab switcher at top: two buttons styled as tabs with bottom border indicator
- Filter bar below tabs: inline layout (flex-wrap for mobile), always visible (not collapsible)
- Table fills remaining space
- "Export PDF" button fixed at bottom-right of content area (not sticky — scrolls with content)
- On Client Report tab: client selector dropdown appears as first filter element

### Settings Page (`/settings`)

**Focal point:** The logo upload area is the primary visual anchor. "Upload Logo" / "Replace" is the primary action focal point.

```
+----------------------------------------------+
| Company Settings              (page heading)  |
+----------------------------------------------+
| Company Logo                                  |
| +--------+                                    |
| | [logo] |  [Upload Logo] / [Replace]         |
| +--------+  [Remove] (text-red-600)           |
| Logo appears on PDF reports.                  |
+----------------------------------------------+
```

- Minimal page: heading + single card with logo upload area
- No logo uploaded: show dashed border placeholder with "Upload Logo" button
- Logo uploaded: show preview thumbnail (max 120px wide) with "Replace" and "Remove" options
- "Remove" button styled as `text-red-600 hover:text-red-700` text link (no bg fill)

### Print Layout (`@media print`)

- Hide: sidebar, bottom tab bar, filter bar, tab switcher, "Export PDF" button, page header
- Show: report table only, full width
- Table borders visible for paper readability
- Company name as print header (inserted via CSS)

Source: D-07 (single page with two tabs), D-08 (print stylesheet), D-06 (sidebar nav item).

## Interaction Contract

### Tab Switching
- Click tab to switch between Company Report and Client Report
- Active tab: `border-b-2 border-blue-600 text-blue-600 font-semibold`
- Inactive tab: `text-gray-500 hover:text-gray-700`
- Tab switch resets table data but preserves date range filter values

### Column Sorting (D-05)
- Click any sortable column header to toggle sort
- First click: sort descending (highest first for numbers, Z-A for text)
- Second click on same column: toggle to ascending
- Click different column: sort descending by new column
- Sort indicator: small chevron arrow (up or down) in `text-blue-600` next to active column header text
- Non-active sortable columns show a subtle up/down double-chevron in `text-gray-300`
- Default sort: outstanding balance descending

### Date Range Filter (D-04)
- Two native `<input type="date">` fields: "From" and "To"
- Default: "From" = 30 days ago, "To" = today
- Changing either field triggers re-fetch via React Query (queryKey includes dates)
- No "Apply" button — filters are live

### "Show Settled Clients" Toggle (D-03)
- Checkbox with label: "Show settled clients"
- Default: unchecked (zero-balance clients hidden)
- Toggling immediately filters the loaded data client-side (no re-fetch needed)

### Client Selector (Client Report Tab)
- Native `<select>` dropdown populated from clients list
- Placeholder option: "Select a client..."
- Selecting a client triggers data fetch for that client's report
- No data shown until a client is selected (show prompt message)

### PDF Download
- "Export PDF" button click triggers authenticated fetch to PDF endpoint
- Button shows loading spinner while PDF generates
- Response is a blob; triggers browser download via programmatic `<a>` click
- Filename pattern: `company-report-YYYY-MM-DD.pdf` or `client-report-{name}-YYYY-MM-DD.pdf`

### "Download" on Detail Pages (D-14)
- Transaction detail page: button labeled "Download Receipt PDF"
- Debt detail page: button labeled "Download Statement"
- Both use same download pattern as report PDF
- Secondary button style: `border border-gray-300 text-gray-700 hover:bg-gray-50`

### Logo Remove Confirmation
- "Remove" button click shows a browser `window.confirm()` dialog
- Confirmation copy: "Remove company logo? It will no longer appear on exported PDFs."
- On confirm: sends DELETE request to logo endpoint, clears preview
- On cancel: no action

## PDF Visual Contract

### Page Setup
- Paper size: US Letter (612 x 792 points)
- Margins: 50pt all sides
- Font: Helvetica (PDFKit built-in) (D-12)

### PDF Header (all report types)
- Company logo: left-aligned, max 80pt wide, max 50pt tall (if logo exists)
- Company name: right of logo (or left-aligned if no logo), Helvetica-Bold 16pt, `#111827`
- Report title below company name: Helvetica 12pt, `#6B7280`
- Date range below title: Helvetica 10pt, `#6B7280`
- Horizontal rule: 0.5pt stroke `#D1D5DB` below header, 10pt gap before table

### PDF Table
- Column headers: Helvetica-Bold 9pt, `#374151`
- Header row: background fill `#F3F4F6`, 0.5pt bottom border `#D1D5DB`
- Body rows: Helvetica 9pt, `#111827`
- Row height: 16pt
- Alternating row background: none (keep clean)
- Column alignment: text left, numbers right
- Headers repeat on every page (D-11)
- Text overflow: constrained to column width, ellipsis for single-line cells

### PDF Footer (D-10)
- "Page X of Y" centered, Helvetica 8pt, `#9CA3AF`
- "Generated: YYYY-MM-DD" right-aligned, Helvetica 8pt, `#9CA3AF`
- Footer at page.height - 40pt

### Company Report PDF Columns
| Column | Width | Align |
|--------|-------|-------|
| Client Name | 180pt | left |
| Total Debts | 100pt | right |
| Total Paid | 100pt | right |
| Outstanding Balance | 120pt | right |

### Client Report PDF Sections
1. Client info block: name, email, phone (Helvetica 10pt)
2. Transactions table: Ref#, Date, Description, Total Amount
3. Per-debt sections: debt summary then payments table (Amount, Date, Method, Reference)

### Receipt PDF (D-13, D-15)
- Company header (logo + name)
- Client info: name, email, phone
- Transaction: reference number, delivered date, description
- Line items table: Item, Qty, Unit Price, Total
- Transaction total (bold, right-aligned)
- Debt status line: "Outstanding: $X.XX" or "Fully Paid"
- Payment history table (if debt exists): Date, Amount, Method, Ref#
- Target: single page. Overflow to second page only if content exceeds.

## Responsive Behavior

### Desktop (md and above)
- Sidebar visible with "Reports" and "Settings" nav items
- Filter bar: single horizontal row
- Report table: full columns visible
- "Export PDF" button inline at bottom-right

### Mobile (below md)
- Bottom tab bar with "Reports" icon
- Filter bar: wraps to multiple rows, each filter takes full width
- Report table: horizontal scroll (`overflow-x-auto`) to preserve column data
- Tab switcher: full-width buttons
- "Export PDF" button: full-width at bottom of content

## Accessibility

- Tab switcher uses `role="tablist"` with `role="tab"` and `aria-selected` on each tab
- Sortable column headers use `aria-sort="ascending"` / `aria-sort="descending"` / `aria-sort="none"`
- Date inputs have associated `<label>` elements
- "Export PDF" button indicates loading state via `aria-busy="true"` and `aria-disabled="true"`
- Logo upload uses `aria-label="Upload company logo"`
- Print stylesheet preserves table semantics for screen readers
