## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip spacing |
| sm | 8px | Form field internal padding (py-2), compact row gaps |
| md | 16px | Default element spacing, form field groups (space-y-4), sidebar link padding |
| lg | 24px | Section padding, header horizontal padding (px-6), card padding (p-6) |
| xl | 32px | Layout column gaps, page vertical padding (py-8) |
| 2xl | 48px | Major section breaks between portal summary blocks |
| 3xl | 64px | Page-level top spacing on portal (full-bleed centering) |

Exceptions:
- Touch targets (sidebar links, table rows, tab bar icons): minimum 44px tall — enforced via `min-h-[44px]` where needed
- Bottom tab bar icons: 56px tall to satisfy mobile touch ergonomics

**Source:** Derived from existing code patterns (`px-6 py-4`, `px-3 py-2`, `gap-4`, `py-8`) — 01-05 STATE.md and AppLayout.tsx / LoginPage.tsx.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Table cell content, form helper text, description paragraphs |
| Label | 14px (text-sm) | 500 (medium) | 1.4 | Form field labels, table column headers, sidebar nav links |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Page titles, modal headings, section headings |
| Display | 24px (text-2xl) | 700 (bold) | 1.2 | Top-level page heading ("Clients"), portal balance figure |

Declared weight set: **regular (400)** and **semibold (600)**. Medium (500) is permitted only for labels — never for decorative emphasis.

**Source:** Extracted from LoginPage.tsx (`text-2xl font-bold`, `text-xl font-semibold`, `text-sm font-medium`, `text-sm` body), AcceptInvitePage.tsx, ForgotPasswordPage.tsx — consistent across all Phase 2 pages.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f9fafb` (gray-50) | Page background (`bg-gray-50`), portal page background |
| Secondary (30%) | `#ffffff` (white) | Cards, modal surface, sidebar background, header background, table background |
| Accent (10%) | `#2563eb` (blue-600) | Reserved elements — see below |
| Destructive | `#ef4444` / red-700 text on red-50 bg | Deactivate client button + confirmation dialog; form validation errors |

Accent (`blue-600`) reserved for:
1. Primary action buttons: "Add Client", "Save Changes", "Send Portal Invite"
2. Active sidebar navigation link (left border indicator + text color)
3. Active bottom tab bar icon
4. Focus ring on form inputs (`focus:ring-blue-500`)
5. Inline hyperlinks (forgot password, portal "Back to" links)

Accent is NOT used for: table row hover states, badges, status labels, secondary buttons, or informational text.

**Borders and dividers:** `border-gray-200` for all card/header/input borders. `border-gray-300` for input fields at rest. Table row dividers use `divide-y divide-gray-200`.

**Status badge colors** (semantic, not accent):
- Active / Open / Confirmed: `bg-green-100 text-green-800`
- Inactive / Written-off: `bg-gray-100 text-gray-600`
- Pending / Awaiting Confirmation: `bg-yellow-100 text-yellow-800`
- Partially Paid: `bg-blue-100 text-blue-700` (distinct from accent — background only)
- Fully Paid: `bg-green-100 text-green-800`

**Source:** LoginPage.tsx, ForgotPasswordPage.tsx, AcceptInvitePage.tsx — consistent blue-600/gray-50/white palette throughout. Status badges derived from FR-03.6, FR-08 (pending/confirmed separation requirement).

## Accessibility Contracts

- All interactive elements reachable by keyboard (Tab order follows visual order)
- Modal traps focus while open; returns focus to trigger on close
- Table rows: `role="button" tabIndex={0}` with `onKeyDown` Enter/Space trigger
- Form errors announced via `aria-live="polite"` region
- Status badges use `aria-label` when color is the only differentiator (e.g., `aria-label="Status: Active"`)
- Bottom tab bar icons: `aria-label` on each tab button (no text visible on mobile)
- Balance "as of [date]": date rendered as `<time dateTime="ISO">` element
- Sidebar nav: `<nav aria-label="Main navigation">` wrapping `<ul>` with `<li>` items
- Minimum contrast: all body text on white/gray-50 meets WCAG AA (gray-700 on white = 9.7:1)
