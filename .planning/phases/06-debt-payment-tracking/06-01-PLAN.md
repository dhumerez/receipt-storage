**Endpoints:**

1. **GET `/:id`** — Debt detail (FR-07.5, D-06)
   - Validate `id` is UUID. Query `debtBalances` view joined with `debts` table (for writeOffReason, transactionId, createdAt) WHERE `debts.id = id AND debts.companyId = companyId`.
   - Join `clients` on `debts.clientId` to get `clients.fullName` as `clientName`.
   - Join `transactions` on `debts.transactionId` to get `transactions.referenceNumber` as `transactionRef`.
   - Separately query `payments` WHERE `debtId = id` ordered by `paidAt DESC` (newest first per D-07). For each payment, join `users` on `payments.recordedBy` to get `users.fullName` as `recordedByName`.
   - Separately query `documents` WHERE `entityType = 'payment' AND entityId IN (payment ids)` to attach docs to each payment.
   - Separately query `documents` WHERE `entityType = 'transaction' AND entityId = debt.transactionId` for transaction documents.
   - Return shape:
     ```json
     {
       "id": "uuid", "companyId": "uuid", "clientId": "uuid", "clientName": "string",
       "transactionId": "uuid", "transactionRef": "TXN-2026-0042",
       "totalAmount": "100.00", "amountPaid": "60.00", "remainingBalance": "40.00",
       "status": "partially_paid", "writeOffReason": null,
       "createdAt": "ISO",
       "payments": [{ "id", "amount", "paidAt", "paymentMethod", "reference", "notes",
                      "status", "recordedByName", "rejectionReason", "createdAt", "documents": [{ "id", "filePath", "originalName", "mimeType", "fileSizeBytes" }] }],
       "transactionDocuments": [{ "id", "filePath", "originalName", "mimeType", "fileSizeBytes" }]
     }
     ```
   - 404 if debt not found or wrong companyId.

2. **POST `/:debtId/payments`** — Record payment (FR-08.1-FR-08.6, D-01-D-04)
   - Apply `uploadMiddleware.array('files', 5)` as route-level middleware.
   - Parse `req.body.data` as JSON (same FormData pattern as transactions.ts POST).
   - Validate with zod schema:
     ```typescript
     const CreatePaymentSchema = z.object({
       amount: z.string().regex(/^\d+(\.\d{1,2})?$/, 'Must be a valid amount'),
       paidAt: z.string().min(1),
       paymentMethod: z.string().min(1).max(100),
       reference: z.string().max(100).optional(),
       notes: z.string().optional(),
     });
     ```
   - Inside `db.transaction()`:
     - SELECT debt WHERE `debts.id = debtId AND debts.companyId = companyId`. 404 if not found.
     - Reject if debt status is `fully_paid` or `written_off` (400: "Cannot record payment on a {status} debt").
     - **Overpayment prevention (FR-08.6):** Query `SUM(amount) FILTER (WHERE status != 'rejected')` from payments WHERE debtId. Compare `existingCents + newPaymentCents > totalDebtCents`. If exceeds, return 400 with `{ error: "Payment would exceed remaining debt balance", maxAmount: fromCents(totalDebtCents - existingCents) }`.
     - **Role-based status (FR-08.4/FR-08.5):** `status = req.user.role === 'owner' ? 'confirmed' : 'pending_approval'`.
     - Insert into `payments` table: `{ companyId, debtId, recordedBy: req.user.sub, status, amount: parsed.data.amount, paidAt: new Date(parsed.data.paidAt), paymentMethod, reference: reference || null, notes: notes || null }`. If owner, set `approvedBy: req.user.sub, approvedAt: new Date()`.
     - **Debt status auto-transition (FR-07.2):** If owner (payment is confirmed), compute new confirmed total. If `newTotal >= totalDebtCents`, update debt status to `fully_paid`. Else if `newTotal > 0`, update to `partially_paid`.
     - **Process uploaded files** (if any): loop `req.files`, call `processFile(file, companyId, 'payments', payment.id)`, insert into `documents` table.
     - **Audit log:** Insert `{ companyId, actorId: req.user.sub, actorRole: req.user.role, action: 'payment_created', entityType: 'payment', entityId: payment.id, newValue: JSON.stringify({ amount, debtId, status }) }`.
     - **Notifications (FR-09.3):** If collaborator (pending_approval), notify all active owners: `{ companyId, recipientId: owner.id, entityType: 'payment', entityId: payment.id, action: 'payment_submitted' }`. Query owners with `SELECT id FROM users WHERE companyId = companyId AND role = 'owner' AND isActive = true`.
   - Return 201 with created payment object.

3. **POST `/:debtId/payments/:paymentId/approve`** — Owner approves pending payment (FR-08.7)
   - Owner-only check: `if (req.user.role !== 'owner') return res.status(403)`.
   - Inside `db.transaction()`:
     - **SELECT FOR UPDATE on debt row:** `await tx.select().from(debts).where(and(eq(debts.id, debtId), eq(debts.companyId, companyId))).for('update')`. 404 if not found.
     - **SELECT FOR UPDATE on payment row:** `await tx.select().from(payments).where(and(eq(payments.id, paymentId), eq(payments.debtId, debtId))).for('update')`. 404 if not found, 400 if `status !== 'pending_approval'`.
     - **Re-validate overpayment:** Query `SUM(amount) FILTER (WHERE status = 'confirmed')` from payments WHERE debtId. If `confirmedCents + thisPaymentCents > totalDebtCents`, return `{ error: 'Approving this payment would exceed the debt total' }`.
     - Update payment: `{ status: 'confirmed', approvedBy: req.user.sub, approvedAt: new Date(), updatedAt: new Date() }`.
     - Update debt status: compute `newTotal = confirmedCents + thisPaymentCents`. If `newTotal >= totalDebtCents` set `fully_paid`, else `partially_paid`.
     - **Audit log:** `{ action: 'payment_approved', entityType: 'payment', entityId: paymentId }`.
     - **Notification:** Notify the payment's `recordedBy` user: `{ entityType: 'payment', entityId: paymentId, action: 'approved' }`.
   - Return 200 with updated payment.

4. **POST `/:debtId/payments/:paymentId/reject`** — Owner rejects pending payment
   - Owner-only check.
   - Validate body: `z.object({ reason: z.string().min(1).max(500) })`.
   - Inside `db.transaction()`:
     - SELECT FOR UPDATE on payment. 400 if not `pending_approval`.
     - Update payment: `{ status: 'rejected', rejectionReason: reason, updatedAt: new Date() }`.
     - **Audit log:** `{ action: 'payment_rejected', entityType: 'payment', entityId: paymentId, metadata: JSON.stringify({ reason }) }`.
     - **Notification:** Notify `recordedBy`: `{ entityType: 'payment', entityId: paymentId, action: 'rejected' }`.
   - Return 200 with updated payment.

5. **POST `/:id/write-off`** — Owner writes off debt (FR-07.6, D-08-D-11)
   - Owner-only check.
   - Validate body: `z.object({ reason: z.string().min(1).max(500) })`.
   - Inside `db.transaction()`:
     - SELECT debt WHERE id AND companyId. 404 if not found.
     - Reject if status is `fully_paid` or `written_off` (400).
     - Update debt: `{ status: 'written_off', writeOffReason: reason, updatedAt: new Date() }`.
     - **Audit log:** `{ action: 'debt_written_off', entityType: 'debt', entityId: id, oldValue: debt.status, newValue: 'written_off', metadata: JSON.stringify({ reason }) }`.
   - Return 200 with updated debt.

6. **POST `/:id/reopen`** — Owner reopens written-off debt (D-10)
   - Owner-only check.
   - Inside `db.transaction()`:
     - SELECT debt WHERE id AND companyId. 404 if not found.
     - Reject if status is NOT `written_off` (400: "Only written-off debts can be reopened").
     - Compute confirmed payment total to determine new status: 0 = `open`, >0 but < total = `partially_paid`.
     - Update debt: `{ status: newStatus, writeOffReason: null, updatedAt: new Date() }`.
     - **Audit log:** `{ action: 'debt_reopened', entityType: 'debt', entityId: id, oldValue: 'written_off', newValue: newStatus }`.
   - Return 200 with updated debt.
